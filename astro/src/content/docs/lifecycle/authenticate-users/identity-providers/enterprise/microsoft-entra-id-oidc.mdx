## Configure OpenID Connect with Microsoft Entra ID

Once you have completed this configuration you may enable an OpenID Connect <InlineUIElement>Login with Microsoft Entra ID</InlineUIElement> button for one or more FusionAuth Applications. See [Microsoft Entra ID - Register An App Quickstart Guide](https://learn.microsoft.com/en-us/entra/identity-platform/quickstart-register-app) as an additional reference.

### Register a New Microsoft Entra ID Application

You will first need to login to the [Azure Portal](https://azure.microsoft.com/en-us/features/azure-portal/).

Once logged in, navigate to <Breadcrumb>Microsoft Entra ID -> App Registrations -> New Registration</Breadcrumb> to create a new Microsoft Entra ID Application.

<img src="/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/azure-openid-connect-register-application.png" alt="Register a new Microsoft Entra ID Application" width="1200" />

Here we have configured our application `Redirect URI`.  If FusionAuth is running at `https://login.fusionauth.io`, this value should be `https://login.fusionauth.io/oauth2/callback`.

<img src="/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/azure-openid-connect-client-id.png" alt="Microsoft Entra ID Client ID and Tenant ID" width="1200" />

Once the application has been created, note the `Application (client) ID` and the `Directory (tenant) ID`.  These will be used respectively as the <InlineField>Client Id</InlineField> value and to construct the <InlineField>Issuer</InlineField> value in your FusionAuth OpenID Connect Identity Provider configuration.

### Create a New Microsoft Entra ID Application Secret

Navigate to <Breadcrumb>Microsoft Entra ID -> App Registrations -> Your Application -> Certificates & secrets -> New client secret</Breadcrumb> to create a new Microsoft Entra ID Application Client Secret.

<img src="/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/azure-openid-connect-client-secret.png" alt="Microsoft Entra ID Client Secret" width="1200" />

Note the `VALUE` of the created client secret.  This will be used as the <InlineField>Client secret</InlineField> value in your FusionAuth OpenID Connect Identity Provider configuration.

### Configure a New FusionAuth OpenID Connect Identity Provider

To create a Microsoft Entra ID Identity Provider return to FusionAuth and navigate to <Breadcrumb>Settings -> Identity Providers</Breadcrumb> and click <InlineUIElement>Add provider</InlineUIElement> and select <InlineUIElement>OpenID Connect</InlineUIElement> from the dialog.

This will take you to the `Add OpenID Connect` panel, and you'll fill out the required fields.

You will need to set the <InlineField>Client authentication method</InlineField> to `HTTP Basic authentication (client_secret_basic)`.

<InlineField>Client Id</InlineField> and <InlineField>Client secret</InlineField> values reference the previously noted Microsoft Entra ID Application's `Application (client) ID`, client secret `VALUE`. The <InlineField>Redirect URL</InlineField> is read only and generated for you based upon the URL of FusionAuth, this value should match the one you configured in your Azure application.

Microsoft Entra ID has implemented a well-known configuration endpoint, so FusionAuth will be able to discover the necessary endpoints using a discovery document by entering the Microsoft Authority URL in the <InlineField>Issuer</InlineField> field. To see the <InlineField>Issuer</InlineField> field, you may need to toggle <InlineField>Discover endpoints</InlineField>.

The Microsoft URL may differ across national clouds, so you will need to review the Microsoft documentation to ensure you have the correct URL for your region. For the Microsoft global Microsoft Entra ID service, the URLs are as follows, where `{tenantId}` is the `Directory (tenant) ID` previously noted while creating our Microsoft Entra ID Application.

- Microsoft Entra ID v1  `https://login.microsoftonline.com/{tenantId}`
- Microsoft Entra ID v2  `https://login.microsoftonline.com/{tenantId}/v2.0`

You may set a <InlineField>Reconcile lambda</InlineField> to map attributes from the JWT provided by Microsoft Entra ID to the FusionAuth user or registration. This is optional. [Learn more about lambdas](/docs/extend/code/lambdas/openid-connect-response-reconcile). You may also modify the <InlineField>Button text</InlineField> or <InlineField>Button logo</InlineField> if desired.

You will need to specify `openid` as a <InlineField>Scope</InlineField>.

<img src="/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/top-of-fusionauth-azure-ad-oidc-config.png" alt="FusionAuth Microsoft Entra ID IdP Configuration" width="1200" />

Scroll down and make sure you enable this Identity Provider for your application. In the following screenshot you will see that we have enabled this login provider for the `Pied Piper` application and enabled `Create registration`.

<img src="/img/docs/lifecycle/authenticate-users/identity-providers/enterprise/bottom-of-fusionauth-azure-ad-oidc-config.png" alt="FusionAuth Microsoft Entra ID IdP Configuration" width="1200" role="top-cropped" />

That's it, now the <InlineUIElement>Login with Microsoft Entra ID</InlineUIElement> button will show up on the login page of our `Pied Piper` application.
